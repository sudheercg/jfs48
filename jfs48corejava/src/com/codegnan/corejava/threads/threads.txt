Thread 

- A thread is an executional path or sequnce in the program. 


Where does a thread exist ?

- When we switch on the computer, a special program called
  OS or operating system is loaded from hard disk to memory(RAM)
  
 - This OS will help us interact with the hardware and the CPU.
 
 - When you try to use any application in your computer, 
   the OS will take that application program from the disk and creates
   an instance of that application in the memory. 
   
- This instance is called a process, 
  process is also known as "Context" of the application. 
  
- Each process is separated from other processes 
  that run on the system. 
  
- A process contains metadata which will have the following: 
   - Process ID or PID
   - files which the application uses for reading and writing of data.
   - code or the program instructions which are going to be
      executed on the CPU
   - heap which contains the data needed for our application.
   
   - main thread : a thread will have its own stack and instruction
     pointer. 
      - stack: A region in the memory where our local variables or
         object references and methods are executed. 
         
      - instruction pointer: It s a pointer which points to 
        the address of the next instruction which the thread
        is going to execute.
        
        
 Process vs Threads:
 ===================
 
 Memory:
  - Process has its own memory
  - Threads will share the memory with other threads in the process
  
 Weight: 
  Process is heavy weight as it takes more time to start and stop and 
  consumes more resources like memory.
  Switching from one process to another process is not fast.(Inter
  process communication or context switching)
  
  Thread is Light weight, takes less time to start and stop.
  Switching from one thread to another thread is faster. (Inter
  thread communication)
  
  
  Thread class: 
  =============
  Thread class is present in java.lang
  
  A Thread can be created in 2 different ways: 
  1. Extendiong the Thread class
  2. Implemening Runnable interface. 
  
  Thread has a method run(): 
  - It contains actual logic that the thread will executed.
  - We must not call run() directly, 
  - we must call start() and JVM internally calls run(() of the thread
  
  start(): 
  It is used to start a thread. 
  It creates a call stack. 
  It calls run() internally
  Executes concurrently with other threads. 
  
  we call start() method only once per thread
  YOu will get IllegalThreadException if you repeatedly call thread
   more than once. 
   
  slee0(milliseconds):
  Pauses the execution of the current thread for specified time. 
  Thread goes into a state called TIMED_WAITING state
  It will hold the lock. 
  after the time thread becomes runnable again. 
  
  
  Interrupted Exception: 
  It indicates that the thread was interrupted while sleeping or
  waiting. 
  During sleep() when another thread calls thread.interrupt().
  
  Thread creation using Runnable interface: 
  ============================================
  - Runnable interface is a interface which contains a single abstract
    method - run(). 
  - It is a functional interface because it has a single abstract method. 
  To create a thread using Runnable interrface:
   1. Create a class which implements Runnable interrace
   2. Override run() method
   3. To use this thread, 
      Create a Thread object.   
      Pass Runnable object to Thread. 
      call start()
      
 Synchronized: 
      Synchronized keyword is used to control access to code by multiple threads.
  - It makes sure that only one thread at a timecan execute the synchronized code
  on the same objects monitor lock.
  It prevents inconsistent data or race coditions when multiple threads
  try to modifiy shared resources.
  
  Avoiding the Race condition using synchronized keyword:
  ------------------------------------------------------
  
  - when a thread enters a synchronized method/block,
  it acquires the lock(monitor) of the object
  other threads trying to execute any synchornized method on the same object
  must wait until the lock is released.
  
  When synchornized section is executed, no other thread can
  interrupt in the middle.
  
  Before releasing the lock, changes made by the thread are flushed to 
  main memory. 
  
  when next thread acquires the lock, it reads the latest updated values
  from the memory.

  Thread Life Cycle:
  -----------------
  
  Every java thread will go through set of states 
  during its life time. 
  The JVM manages the thread life cycle. 
  
 NEW STATE: 
 ==========
 The thread object is created,  but start() is not yet
 called.  
 
 RUNNABLE STATE: 
 ---------------
 The thread is eligible to run, 
 it may be executing on a CPU
 
 BLOCKED STATE: 
 When a thread is waiting to acquire
 a lock 
 When it tries to enter a synchronized block
 
 WAITING STATE: 
 A thread is said to be in WAITING state when
 it indefinitely waits for another thread by calling
 wait() or join().
 
TIMED WAITING: 
A thread is said to be in TIMED_WAITING stATE when
sleep(milliseconds), join(millisecond)

TERMINATED: 
A thread is said to be in a terminated state  when 
run() has completed . 
  
  
wait(), notify(), notifyAll()

Producer: Produces the message(writing the message)
Consumer: Consumes the message(reading the message)

Producer will produce the new message only when 
consumer consumes the message.

Consumer can consume the message only when the producer
produces a new message. 

Whenever the consumer consumes message, it informs the 
producer that the message has been consumed. 

Whenever the producer produces the message, it informs
the consumer that the message has been produced.
  
  wait():
  wait() method is defined in Object class
  It causes the current thread to pause (go into waiting stage)
  until another thread calls notify() or notifyAll()
  on the same object. 
  
  _ It must be called inside a synchronized method 
  when a thread calls wait, it releases the lock so that other threads can 
  acquire the lock on this object. 
  
  
  
  notify():
  -wakes up one waiting thread which is waiting for the same objects lock.
  -does not release the lock immediately. 
  - the lock will be released only when the current thread exits the synchronized block. 
  - It is useful when you want to notify a single thread to resume its work.
  
  
  
  notifyAll()
  -  Wakesup all the threads that are waiting for the same objects lock . 
  
  wait() , notify(), notifyAll methods belong to Object class.
  They are not the methods of Thread class.
